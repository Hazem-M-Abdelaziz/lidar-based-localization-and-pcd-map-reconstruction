# lidar-based-localization-and-pcd-map-reconstruction

## 🗺️ Project Overview

This project focuses on the processing of ROS 2 bag files, including sensor data extraction and preprocessing, LiDAR-based pose estimation and localization, and the construction of a 3D point cloud map from recorded LiDAR scans.


## 📂 Project Hierarchy

The following is the folder structure of the project, highlighting key files and directories:

```plaintext
Project_Root/
├── lidar_pose_estimator/                        # ROS2 Pkg directory
│   ├── build/                                       # ROS2 built files for the package
│   ├── config/                                      # Configurations folder
|       ├── micropolisConfig.rviz                    # RViz config file passed when visualizing data.
│   ├── include/lidar_pose_estimator/                # include folder containing created header files.
|       ├── evaluation_node.hpp                      # Header file for evaluation node.
|       ├── lidar_pose_estimator_node.hpp            # Header file for lidar pose estimator node.
│   ├── install/                                     # ROS2 Install files for the package
│   ├── log/                                         # ROS2 log files for the package 
│   ├── src/                                         # ROS2 source code folder for the package 
|       ├── evaluation_node.cpp                      # Source code file for evaluation node.
|       ├── lidar_pose_estimator_node.cpp            # Source code file for lidar pose estimator node.
│   ├── CMakeLists.txt                               # CMakeLists file for building the package
│   ├── package.xml                                  # package.xml file for building the package
├── tmp/                                         # Directory for output files and qos argument file (Note that all generated files are based on bag_07 file)
│   ├── lidar_map.pcd                                # point cloud map of the recorded scene generated by lidar_pose_estimator node
│   ├── lidar_map_eval.pcd                           # point cloud map of the recorded scene generated by evaluation node
│   ├── lidar_tum.txt                                # TUM format file generated by lidar_pose_estimator node
│   ├── lidar_tum_eval.txt                           # TUM format file generated by evaluation node
│   ├── pose_error.csv                               # csv file created by evaluation node for the recorded bag file
│   ├── qos_override.yaml                            # file for bag file playing arguments
└── LICENSE                                          # Project license
└── README.md                                        # Project documentation
```

## ⚙️ Project Setup
0. Before running the program, you can check bag file info using:
   ```
   ros2 bag info <path_to_bag_file>
   ```
   you're expected to see output like this:
   ```
   Files:             /home/hazem-m-abdelaziz/Downloads/micropolis_bag_07.mcap
    Bag size:          2.5 GiB
    Storage id:        mcap
    ROS Distro:        unknown
    Duration:          50.930368413s
    Start:             Jan 27 2026 14:00:12.132635229 (1769515212.132635229)
    End:               Jan 27 2026 14:01:03.063003642 (1769515263.063003642)
    Messages:          9415
    Topic information: Topic: /fixposition/odometry_enu | Type: nav_msgs/msg/Odometry | Count: 4483 | Serialization Format: cdr
                       Topic: /fixposition/odometry_llh | Type: sensor_msgs/msg/NavSatFix | Count: 4482 | Serialization Format: cdr
                       Topic: /iv_points_fusion | Type: sensor_msgs/msg/PointCloud2 | Count: 450 | Serialization Format: cdr
    Service:           0
    Service information: 
   ```
2. Clone the pkg folder inside your `<ros2_ws>` directory locally.
3. rebuild your workspace with new package added, (you can remove build and install files of the pkg to avoid conflicts).
   ```bash
   rm -rf build/lidar_pose_estimator install/lidar_pose_estimator
   colcon build --packages-select lidar_pose_estimator
   ```
4. Now you can run both nodes using:
   ```bash
   ros2 run lidar_pose_estimator lidar_pose_estimator_node   --ros-args   -p map_pcd_path:=desired_output_for_pcd_map_file_path_by_estimator_node>   -p tum_path:=:=<desired_output_path_for_TUM_estimator_node_file_output>
   ```
   for the lidar pose estimator node, and
   ```bash
   ros2 run lidar_pose_estimator evaluation_node   --ros-args   -p output_pcd_path:=<desired_output_for_pcd_map_file_path_by_evaluation_node>   -p output_tum_path:=<desired_output_path_for_TUM_evaluation_node_file_output>
   ```
   for the evaluation node.

5. You can also check the node overview, for example:
   ```bash
   ros2 node info /lidar_pose_estimator
   ```
   expected output:

6. Opening RViz2 interface to visualize preocessing data
   ```bash
    rviz2 -d <path_to_rviz_configuration_file>
   ```
7. Playing the bag file
   ```bash
   ros2 bag play <path_to_bag_filep>   --clock --rate 0.1   --qos-profile-overrides-path <path_to_qos_override_file>
   ```
